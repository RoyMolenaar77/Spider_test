De code staat nu in losse stapjes;

 

Eerst de code achter

knop 1, leeg maken temp tabellen, bewaren import status + laatste datum van producten die bij de vorige runs al zijn verwerkt.

Dan

Knop 2, import index file1 (100mb), sql dump de data in MSSQL (template word via c# code aangeroepen)

Knop 3, import index file2

Knop 4, voorbereiden detail import MSSQL tabel icecat_productsmap, o.a. if path contains (‘/NL’) dan languageID kolom invullen, etc..

Knop 5, op basis van Icecat_Productsmap, alle product detail files downloaden en SQL BULK inserten (xml templates), SQL Job dt_replicate aanroepen.

Nu is het volledige import proces uitgevoerd.

 

 

De ‘echte’ logica zit in

 

C:\userfolder\ICECAT\Production\Main IceCat Index 2 SQL\productsmap.xml

C:\userfolder\ICECAT\IceCat_ProductDetails\Productdetailsmap.xml

 

De vb scripts in de laatste directory zijn een voorbeeld om vanuit een cmd prompt de sql bulk import uit te voeren.

Deze scripts worden niet meer echt gebruikt en kunnen dus iets gedateerd zijn. (volgens mij werken ze nog gewoon prima)

 

Je wil de app gewoon geheel opnieuw schrijven, dit is het eerste en enige C# programma wat ik ooit heb geschreven.

De SQL BULK logica is volgens mij redelijk goed, ik adviseer om hiervoor het wiel niet geheel opnieuw te vinden, wellicht moeten er hier en daar nog velden/relaties worden toegevoegd, maar dit kan gewoon in de xml templates.
